Use Cases(suggested)

  • oF's core libraries deal with Multimedia, 2D and 3D graphics, images, video and sound. oF is good for developing new media projects for onsite installations and also rich multimedia desktop or mobile apps
  • It works with the C++ language which is compiled to native machine code for the target platform, whch results in very performant fast applications that maximize the latest harware.
  • It is not Web first (it is possible to transpile to asm.js Emscripten but this is very new and not eveything works)
  • Great for interaction prototyping!

Note there are two build configurations, "Debug" and "Release".

Debug is useful when developing your project, as it will provide the most information about where and why something crashed.

Release is useful when you're done developing your project. Release will create a smaller, faster app -- but it won't give you much information if it crashes.

Now, Lets have a look at some of the important folders in the oF folder:

  • Examples
  • Apps
  • Addons

Examples

  • 3D: Examples for 3D graphics and 3D math
  • addons: Examples of various addons, extensions of oF core API i.e. Kinect, OpenCV, OSC
  • communication: Examples for communication with hardware vi Serial port. i.e. talk to an Arduino
  • empty: contains emptyExample. Copy this and use as a starting template
  • events: Examples for dealing with events such as keypress, custom events and timers
  • GL: Advanced Examples for 2D and 3D graphics, FBOs, Shaders etc..

Examples(Cont)

  • graphics: Examples of 2d graphics
  • gui: Examples using the built in GUI
  • math: math examples including a simple particle system, vector math and noise
  • sounds: Examples for playing, recording and basic generation of sounds
  • util: Examples useful topic such as working with directories, saving and loading files, threads etc..
  • video: Examples for playing, recording and processing video. Also how to use the webcam

File structure of a Project

Lets look at the file strucure of oFworkshop/emptyExample

This is a bare minimum project

The bin folder is where our executable or binary file will live once we compile the program

The bin/data folder is where we can include assets for our program such as fonts, images, sounds, movies etc...

The src folder is where our code lives

At minimum there should be main.cpp , ofApp.cpp and ofApp.h

we will walk through each of these files in just a bit

Main.cpp

main.cpp

  • In C++ each project must have a .cpp file with a main() function defined
  • There's not too much to change here, except:
  • ofSetupOpenGL(1024,768, OF_WINDOW);
  • We can change OF_WINDOW to OF_FULLSCREEN to fullscreen the application
  • We can change the size by altering 1024,768

ofApp.h

ofApp.h

  • In the header file we define the structure of our program
  • #pragma once is a compiler directive, include this in all header files
  • #include "ofMain.h" imports all of the core classes from oF
  • once oF is imported we have ofBaseApp available
  • We are defining something called a class named ofApp for our program based on ofBaseApp
  • This is called inheritance
  • Inside our new ofApp class we have also inherited some functions..

ofApp.h(cont)

  • The most important functions are setup(), update() and draw()
  • These functions are the bare minimun required
  • Some other functions such as mouseMoved(int x, int y) are provided, which we may not use
  • In the header file we simply declare the name of classes function and variables for our program
  • We write the code for these in ofApp.cpp

ofApp.cpp

ofApp.cpp

  • The ofApp.cpp file contains the definitions of all functions declared in ofApp.h
  • Lets go through the primary functions
  • setup() is called first and runs only once
  • update() is called every frame before draw()
  • draw() is called every frame all drawing functions should be called here
  • by default the program runs at 60FPS

setup()

  • Some examples of things we may put here:
  • ofSetFrameRate(30); //reduce the framerate to 30FPS
  • ofSetFullscreen(1); //set fullscreen to true
  • Things you only need to do once

update()

  • Some examples of things we may put here:
  • Update a variable before redering it to the screen in draw()
  • read data from the network
  • Things you need to happen before drawing the results to the screen

draw()

  • Some examples of things we may put here:
  • ofDrawCircle(200,400,80); // draw a shape
  • ofDrawBitmapString("hello", 300, 300); //draw some text
